# v3.19 — Oracle-Anchored Pool (Phase 2)

**Status:** Code complete. Deploy when MVP pool is live and stable.
**Depends on:** v3.18 deployed + pool initialized + users active

---

## Goal

Replace manual LP price management with an automatic oracle-anchored system:
- Admin sets USDC/COP rate once daily (or on big moves)
- Smart contract rejects swaps that push pool >2% off the real rate
- Backend keeper auto-rebalances the pool when it drifts

---

## What Gets Deployed

All Phase 2 contracts are already written and tested in the codebase. No new code needed — just deploy and wire up.

| Contract | Location | Purpose |
|---|---|---|
| `PriceFeedManager` | `src/contracts/oracles/` | Oracle registry (Chainlink-compatible) |
| `ManualPriceAggregator` | `src/contracts/oracles/` | IAggregatorV3 drop-in, admin sets price |
| `ConvexoHookDeployer` | `src/contracts/hooks/` | CREATE2 factory for ConvexoPoolHook |
| `ConvexoPoolHook` | `src/contracts/hooks/` | New pool hook — KYC + price band + rebalance |

## Deploy Script

Create `script/DeployPhase2.s.sol`:

```solidity
// Deploys: ManualPriceAggregator, PriceFeedManager,
//          ConvexoHookDeployer, ConvexoPoolHook
// Wires:   PriceFeedManager.setPriceFeed(USDC_COP, aggregator, 26h)
// Sets:    ManualPriceAggregator.setPrice(420000000000)  // 4200 COP/USDC
// Grants:  ConvexoPoolHook.grantRole(KEEPER_ROLE, keeperEOA)
```

Key difference from v3.18 deploy:
- `ManualPriceAggregator` + `PriceFeedManager` are **deterministic** (same address all chains)
- `ConvexoPoolHook` is **chain-specific** (mined salt for bits 0x1A80)

## Deploy Checklist

### Pre-flight
- [ ] v3.18 pool live and used by real users
- [ ] Keeper EOA funded with ETH (for gas on Base/Unichain)
- [ ] Backend env vars ready: `KEEPER_PRIVATE_KEY`, `CONVEXO_HOOK_ADDRESS`, etc.
- [ ] Initial reserve amounts ready: ~5,000 USDC + ~21,000,000 ECOP

### Deploy Phase 2 contracts
```bash
KEEPER_ADDRESS=<keeper_eoa> \
forge script script/DeployPhase2.s.sol \
  --rpc-url $RPC_URL --private-key $ADMIN_PRIVATE_KEY --broadcast
```

### Fund the hook reserve
```bash
# Approve tokens first
cast send <USDC> "approve(address,uint256)" <CONVEXO_HOOK> 5000000000 --private-key $ADMIN_PRIVATE_KEY
cast send <ECOP> "approve(address,uint256)" <CONVEXO_HOOK> 21000000000000000000000000 --private-key $ADMIN_PRIVATE_KEY

# Deposit reserve
cast send <CONVEXO_HOOK> "depositReserve(address,uint256)" <USDC> 5000000000 --private-key $ADMIN_PRIVATE_KEY
cast send <CONVEXO_HOOK> "depositReserve(address,uint256)" <ECOP> 21000000000000000000000000 --private-key $ADMIN_PRIVATE_KEY
```

### Re-initialize pool with new hook
The pool must be created fresh with ConvexoPoolHook (V4 hook address is part of PoolKey — you can't swap hooks on an existing pool).

Options:
1. **Migrate liquidity** — remove all LP from PassportGatedHook pool, initialize new pool with ConvexoPoolHook, LPs re-add
2. **Run both pools** — deprecate the old one by removing liquidity over time

```bash
HOOK_ADDRESS=<convexo_pool_hook> \
TOKEN0=<usdc> TOKEN1=<ecop> \
forge script script/InitializePool.s.sol --broadcast
```

### Configure backend
Add Railway env vars:
```
KEEPER_PRIVATE_KEY=0x...
CONVEXO_HOOK_ADDRESS=0x...
USDC_ADDRESS=0x...
ECOP_ADDRESS=0x...
MANUAL_PRICE_AGGREGATOR_ADDRESS=0x...
POOL_KEEPER_CHAIN_ID=8453
```

Backend pool keeper starts automatically — polls every 5 min, calls `rebalance()` when `deviationBps > bandBps`.

### Set initial price
```bash
# 4200 COP per USDC (adjust to current market rate)
cast send <MANUAL_PRICE_AGGREGATOR> \
  "setPrice(int256)" 420000000000 \
  --private-key $ADMIN_PRIVATE_KEY
```

## Ongoing Operations

### Admin sets price (daily or on big moves)
```bash
# Via backend API (recommended — also updates DB + Redis)
curl -X POST /admin/rates \
  -H "Authorization: Bearer $JWT" \
  -d '{"pair": "USDC-COP", "rate": 4200, "source": "manual"}'
# Backend automatically calls ManualPriceAggregator.setPrice() on-chain

# Or directly on-chain
cast send <MANUAL_PRICE_AGGREGATOR> "setPrice(int256)" <NEW_PRICE>
```

### Monitor pool health
```bash
# Via backend API
GET /pool/status

# Or on-chain
cast call <CONVEXO_HOOK> \
  "getPoolPriceStatus((address,address,uint24,int24,address))" \
  "(<currency0>,<currency1>,500,10,<hook>)"
# Returns: (poolSqrt, oracleSqrt, deviationBps, needsRebalance)
```

### Monitor reserve levels
```bash
cast call <USDC> "balanceOf(address)" <CONVEXO_HOOK>
cast call <ECOP> "balanceOf(address)" <CONVEXO_HOOK>
# Alert when < 20% of initial reserve
```

## Phase 3 — Chainlink (after Phase 2 is stable)

Zero contract changes. Single call:

```bash
cast send <PRICE_FEED_MANAGER> \
  "setPriceFeed(uint8,address,uint256)" \
  0 <CHAINLINK_USDC_COP_ADDRESS> 3600 \
  --private-key $ADMIN_PRIVATE_KEY
```

`ManualPriceAggregator` becomes unused. `PriceFeedManager` now reads from Chainlink automatically.

See `docs/oracles/ORACLES_REFERENCE.md` for Chainlink feed addresses and Proof of Reserves roadmap.
